# Contributing to GestureX Racing

First off, thank you for considering contributing to GestureX Racing! 🎉

## How Can I Contribute?

### 🐛 Reporting Bugs

Before creating bug reports, please check the existing issues to avoid duplicates. When you create a bug report, include as many details as possible:

- **Use a clear and descriptive title**
- **Describe the exact steps to reproduce the problem**
- **Provide specific examples**
- **Describe the behavior you observed and what you expected**
- **Include screenshots if possible**
- **Note your browser version and OS**

### 💡 Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion:

- **Use a clear and descriptive title**
- **Provide a detailed description of the suggested enhancement**
- **Explain why this enhancement would be useful**
- **List some examples of how it would work**

### 🔧 Pull Requests

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Test your changes thoroughly
4. Update documentation if needed
5. Write a clear commit message
6. Submit a pull request

### 📝 Code Style

- Use meaningful variable names
- Comment complex logic
- Follow existing code patterns
- Keep functions focused and small
- Use ES6+ features where appropriate

### 🎯 Areas We'd Love Help With

- **New Gesture Controls** - Add more hand gestures
- **Additional Cars** - More 3D vehicle models
- **Track Variety** - Different racing environments
- **Sound Effects** - Engine sounds, collision effects
- **Multiplayer** - Race against friends
- **Mobile Support** - Touch controls for mobile devices
- **Performance** - Optimization improvements
- **Accessibility** - Making the game more accessible

### 🚀 Getting Started

```bash
# Clone your fork
git clone https://github.com/yourusername/gesturex-racing.git

# Create a branch
git checkout -b feature/my-new-feature

# Make your changes
# ...

# Commit your changes
git commit -m "Add some feature"

# Push to your fork
git push origin feature/my-new-feature

# Open a Pull Request
```

### ✅ Testing Checklist

Before submitting a PR, make sure:

- [ ] The game runs without errors
- [ ] All existing features still work
- [ ] New features are documented
- [ ] Code follows the existing style
- [ ] No console errors
- [ ] Tested on multiple browsers (Chrome, Firefox, Edge)

## Code of Conduct

Be respectful and considerate. We're all here to build something awesome together!

## Questions?

Feel free to open an issue with your question or reach out to the maintainers.

Thank you! 🙏
